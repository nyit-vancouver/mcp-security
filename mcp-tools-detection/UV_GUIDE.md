# uv Guide for MCP Tool Detection System

## 📘 What is uv?

**uv** is a modern Python package manager written in Rust that's **10-100x faster** than pip. It combines the functionality of pip, pip-tools, virtualenv, and more into a single, blazingly fast tool.

### Key Benefits
- ⚡ **10-100x faster** than pip
- 🔒 **Lock files** for reproducible builds (`uv.lock`)
- 🐍 **Python version management** built-in
- 📦 **Single tool** for all package management needs
- 🚀 **Zero Python dependency** - works standalone

## 🛠️ Installation

### Install uv

**macOS/Linux:**
```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

**Windows:**
```powershell
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"
```

**With pip (if you have Python):**
```bash
pip install uv
```

**Verify installation:**
```bash
uv --version
```

## 📁 Project Structure with uv

```
mcp-tools-detection/
├── .python-version        # Python version (3.11)
├── pyproject.toml        # Project config & dependencies
├── uv.lock               # Locked dependencies (auto-generated)
├── .venv/                # Virtual environment (auto-created)
├── app.py                # Your Python code
└── ...
```

### Key Files

**`.python-version`**
- Specifies Python version for this project
- uv automatically downloads it if not installed
- Change version: `echo "3.12" > .python-version`

**`pyproject.toml`**
- Modern Python project configuration
- Replaces `requirements.txt`
- Contains dependencies, metadata, and settings

**`uv.lock`**
- Exact versions of all dependencies
- Ensures reproducible builds
- Commit to version control
- Auto-generated by uv

## 🚀 Quick Start Workflow

### 1. First Time Setup

```bash
# Clone project
cd mcp-tools-detection

# Install dependencies (creates .venv automatically)
uv sync
```

This single command:
1. Creates `.venv` directory
2. Downloads Python 3.11 if needed
3. Installs all dependencies from `pyproject.toml`
4. Creates `uv.lock` file

### 2. Run the Application

```bash
# Run with uv (no activation needed!)
uv run python app.py

# Or activate manually and run
source .venv/bin/activate  # Linux/Mac
.venv\Scripts\activate     # Windows
python app.py
```

## 📦 Managing Dependencies

### Add New Packages

```bash
# Add runtime dependency
uv add requests

# Add development dependency
uv add --dev pytest black ruff

# Add optional dependency group
uv add --optional database sqlalchemy

# Add specific version
uv add "flask>=3.0.0,<4.0.0"
```

**What happens:**
- Package installed in `.venv`
- `pyproject.toml` updated
- `uv.lock` regenerated

### Remove Packages

```bash
# Remove dependency
uv remove requests

# Remove dev dependency
uv remove --dev pytest
```

### Update Dependencies

```bash
# Update all dependencies to latest compatible versions
uv lock --upgrade

# Update specific package
uv lock --upgrade-package flask

# Sync environment after update
uv sync
```

### View Dependencies

```bash
# Show dependency tree
uv tree

# Show outdated packages
uv lock --upgrade --dry-run
```

## 🐍 Python Version Management

### Check Current Version

```bash
cat .python-version
# Output: 3.11
```

### Change Python Version

```bash
# Set to Python 3.12
echo "3.12" > .python-version

# Rebuild environment with new version
uv sync
```

uv automatically:
1. Downloads Python 3.12 if not installed
2. Creates new `.venv` with Python 3.12
3. Reinstalls all dependencies

### Supported Version Syntax

```bash
# Specific version
echo "3.11.7" > .python-version

# Major.minor (uses latest patch)
echo "3.11" > .python-version

# Latest stable
echo "3" > .python-version
```

## 🔧 Advanced Usage

### Development Dependencies

Add tools you need for development but not production:

```bash
uv add --dev pytest black ruff mypy
```

In `pyproject.toml`:
```toml
[project.optional-dependencies]
dev = [
    "pytest>=7.4.0",
    "black>=23.0.0",
    "ruff>=0.1.0",
]
```

Install dev dependencies:
```bash
uv sync --extra dev
```

### Multiple Dependency Groups

```toml
[project.optional-dependencies]
dev = ["pytest", "black"]
database = ["sqlalchemy", "psycopg2"]
all = ["pytest", "black", "sqlalchemy", "psycopg2"]
```

```bash
# Install specific group
uv sync --extra database

# Install multiple groups
uv sync --extra dev --extra database

# Install all groups
uv sync --all-extras
```

### Export for Legacy Tools

If you need `requirements.txt` for Docker or CI:

```bash
# Generate requirements.txt
uv pip compile pyproject.toml -o requirements.txt

# With hashes for extra security
uv pip compile pyproject.toml -o requirements.txt --generate-hashes
```

### Clean Rebuild

```bash
# Remove virtual environment
rm -rf .venv

# Rebuild from scratch
uv sync
```

## 🐳 Docker Integration

### Dockerfile with uv

```dockerfile
FROM python:3.11-slim

# Install uv
COPY --from=ghcr.io/astral-sh/uv:latest /uv /usr/local/bin/uv

WORKDIR /app

# Copy dependency files
COPY pyproject.toml uv.lock ./

# Install dependencies
RUN uv sync --frozen --no-cache

# Copy application
COPY . .

# Run with uv
CMD ["uv", "run", "python", "app.py"]
```

### Alternative: Use requirements.txt in Docker

```dockerfile
FROM python:3.11-slim

WORKDIR /app

# Generate requirements.txt locally first
COPY requirements.txt .
RUN pip install -r requirements.txt

COPY . .
CMD ["python", "app.py"]
```

## 🔍 Troubleshooting

### "uv: command not found"

Add uv to your PATH:
```bash
# Linux/Mac
echo 'export PATH="$HOME/.cargo/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

# Windows
# Add %USERPROFILE%\.cargo\bin to PATH
```

### "Python version not found"

uv will download automatically, but you can force install:
```bash
uv python install 3.11
```

### Virtual environment issues

```bash
# Clean rebuild
rm -rf .venv uv.lock
uv sync
```

### Dependency conflicts

```bash
# See resolution details
uv lock --verbose

# Force specific version
uv add "package==1.2.3"
```

## 🎯 Best Practices

### ✅ DO

- Commit `pyproject.toml` and `uv.lock` to version control
- Use `uv sync` after pulling changes
- Use `uv run` to run scripts (auto-activates environment)
- Keep `uv.lock` up-to-date with `uv lock --upgrade` periodically
- Use version constraints in `pyproject.toml` (e.g., `>=3.0.0,<4.0.0`)

### ❌ DON'T

- Don't mix pip and uv in the same project
- Don't manually edit `uv.lock`
- Don't commit `.venv/` directory
- Don't forget to run `uv sync` after changing dependencies

## 💡 Tips & Tricks

### Faster First Install

```bash
# Use cached packages
uv sync --link-mode=copy
```

### Check What Will Be Installed

```bash
# Dry run
uv sync --dry-run
```

### Offline Mode

```bash
# Use only cached packages
uv sync --offline
```

### Custom Package Index

Edit `pyproject.toml`:
```toml
[tool.uv]
index-url = "https://pypi.org/simple"
extra-index-url = ["https://custom-index.com/simple"]
```
